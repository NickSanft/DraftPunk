Spike S1b — y-partykit + Cloudflare Durable Objects

Validates: (a) two browsers sync via PartyKit, (b) late joiners get full state for free, (c) state survives client refresh and server restart.

How to run:
  1. From spikes/s1b-partykit/, run npm install once, then npm run dev.
  2. Open http://localhost:1999/ in two browser windows. Any combination — PartyKit doesn't have S1's BroadcastChannel confusion. Click Join in both.
  3. Click Add stroke in either; it appears in the other within ~100ms.
The tests that S1 couldn't do:
  1. Late-joiner: add 5 strokes in window A, then open a NEW third window — it should immediately receive all 5 strokes.
  2. Server-restart persistence: stop the server (Ctrl+C in the terminal), start it again, refresh both windows. Strokes should still be there (Durable Object snapshot persistence).
  3. All-clients-disconnected persistence: close both windows, wait 30s, reopen — strokes still there.
